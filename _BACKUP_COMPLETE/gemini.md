## 📐 Структура страницы страны v3.0 (СТРОГО СОБЛЮДАТЬ)

1. `<nav>` — логотип слева, «← Все страны» справа
2. `<h1>` — название страны, **крупно, НЕ В КАПСЛОКЕ**
3. Кнопка «Хочу туда»
4. Hero-фото на всю ширину
5. Свободный контент: абзацы и фотографии чередуются в произвольном порядке
6. Разделы-регионы: заголовок + фото(s) + текст
7. `<footer>` — стандартный
8. `#modal` — расширенная форма с автозаполнением страны

## 🚀 Процесс верстки памяток (СТРОГИЙ ПОРЯДОК)

1. **Text First** — берем исходный `.txt` из папки `content_extracted`. Никакой "чистой" верстки без текста.
2. **Header Structure** — сначала выписываем все заголовки и подзаголовки в текстовом виде.
3. **Approval** — утверждаем структуру заголовков.
4. **Layout** — после утверждения переносим в `template_memo.html`.

## 📋 Правила работы с текстом

### ❌ Проблема: ИИ искажает текст
- Сокращает или перефразирует
- «Улучшает» стиль
- Теряет часть предложений

### ✅ Решение: давать текст через файл

**Способ 1 — файл .txt (РЕКОМЕНДУЕТСЯ)**
Создай файл `[страна]_content.txt` по формату:
```
СТРАНА: Египет
SLUG: egypt
HERO_PHOTO: egypt_hero.jpg

ТЕКСТ:
Первый абзац...

Второй абзац...

ФОТО: egypt_photo.jpg

Третий абзац...

РЕГИОН: Хургада
ФОТО: hurghada.jpg
Текст о Хургаде...
```
Скажи: «возьми текст из файла egypt_content.txt и вставь дословно»

## ⚖️ Правила поведения

1. **Russian Language Only** — все ответы только на русском.
2. **No Auto-Run** — все команды передаются пользователю для ручного запуска.
3. **Template First** — новую страницу всегда копировать из соответствующего шаблона.
4. **Текст дословно** — никогда не перефразировать и не сокращать текст пользователя.
5. **Параграфы раздельно** — каждый смысловой абзац = отдельный `<p>`.
6. **Стиль не трогать** — CSS-классы не меняются при правках контента.
7. **PAGE_COUNTRY** — в каждой странице заменять эту переменную на название страны.
9. **NO CAPS** — использование Caps Lock запрещено в заголовках `<h1>`, `<h2>` и основном тексте. 
10. **NO PHOTO PLACEHOLDERS** — все строки типа «Фото: tild...» удаляются.
11. **MEMO PAGES** — на страницах памяток правила соблюдать особенно строго.
12. **AUTOMATION LOG** — любое массовое изменение фиксировать в `SCRIPTS.md`.
13. **ROOT CLEANUP** — корень проекта должен содержать только основные HTML и папки. Скрипты и временные файлы удаляются.
